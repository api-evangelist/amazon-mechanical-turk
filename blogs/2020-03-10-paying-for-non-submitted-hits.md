<p>Mechanical Turk is designed to require an Assignment ID in order for a Requester to issue a payment to a Worker. However, as an Assignment ID is only generated when a Worker successfully submits a HIT to a Requester for review, it is not currently possible to issue a payment to a Worker who was unable to submit a HIT to a Requester for review. However, Workers will occasionally contact Requesters for assistance when they have completed the work requested, but were unable to successfully submit the work to the Requester through the Worker UI. We have set up the following HIT template to allow Requesters to create a simple HIT for these Workers, so the Requester can pay the Worker should they choose to do so.</p><p>For this HIT, we will be creating a simple layout where the Worker can accept the HIT, enter their Worker ID and submit the HIT for approval. When setting up the HIT, you’ll have the option to either set the full amount you intend to pay, or set it for $0 enabling you to bonus them the reward amount afterwards.</p><p><strong>Please note</strong>: The reward amounts you chose are still subject to the Mechanical Turk fee structure, so please be sure to review your cost estimates before publishing your HIT (https://requester.mturk.com/pricing).</p><p>While this walk through is designed to create a HIT for one Worker, it can also be used to create a HIT for multiple Workers as well. You will need to assign all the Workers the same qualification and set the number of assignments for the HIT to the total number of Workers you are wanting to pay.</p><h3>Custom Qualification</h3><p>The first thing you want to do is create a Custom Qualification to grant to the Worker in order to make sure they are the only one that can accept and submit the HIT.</p><p>To create a qualification using the Requester UI:</p><p>1. Sign in here, <a href="https://requester.mturk.com/qualification_types">https://requester.mturk.com/qualification_types</a></p><p>2. Click the ‘Create New Qualification Type’ button and give the qualification a name and description.</p><p>3. Click Create and the qualification will appear in a list on the page. You can create as many different qualifications as you like.</p><p>To provide a Custom Qualification to a Worker who has not submitted an assignment to you using the Website UI:</p><p>1. Go to the Worker’s page by adding their Worker ID at the end of this URL: <a href="https://requester.mturk.com/workers/">https://requester.mturk.com/workers/</a></p><p>a. Example: <a href="https://requester.mturk.com/workers/">https://requester.mturk.com/workers/</a>ABCDEF123456</p><p>2. Click on ‘Assign Qualification Type’ button.</p><p>3. Choose the Qualification you want to assign from the pop-up list.</p><p>4. Once you check the box for the qualification, you will be prompted to assign a score from 1 to 100, based on how you rate that particular Worker.</p><h3>Create the HIT</h3><p>1. Click on the ‘Create’ tab and select ‘New Project’.</p><p>2. Choose the ‘Survey Link’ category and click ‘Create Project’.</p><p>3. If you are making a HIT for only one Worker, change the Properties screen to look like the following. If you are making a HIT for multiple Workers, you can edit the properties however you need to, primarily the title and the number of respondents (second highlighted item).</p><figure><img alt="" src="https://cdn-images-1.medium.com/max/692/1*NlaAoYQQdpVF1ytyS7tcCg.png" /></figure><p>a. The bottom highlighted area is where you would add the Custom Qualification that you created to ensure only that Worker can accept the HIT. Click ‘Add another criterion’ and select your qualification. Set the integer amount to what you assigned the Worker.</p><p>b. N<strong>ote</strong>: If you want to pay Workers through a bonus, set the reward to $0 (first highlight item). However, if you want to pay them through this HIT, set the reward amount to the amount you want to pay.</p><p>4. You can click the ‘Publish Batch’ button next to the project you just created and follow the steps to publish it to the site.</p><h3>Payment</h3><p>If you set the amount you want to pay the Worker as the reward amount, all you have to do is approve the submission. However, if you set a $0 amount, you will need to grant a bonus after you have approved the submission.</p><p>To do this, go to the Manage tab and click on the Workers link at the top of the page. Click on the Worker ID that you want to open that Worker’s page. (You can also access this page by clicking on the Worker’s ID link on the Review Results page or in the CSV file.) <br /> <br />Select the Bonus Worker button on the right side of the page. Enter the amount and provide the assignment ID. To locate the assignment ID, download the batch results and locate the Worker’s ID, the corresponding assignment ID is in the same row. Keep in mind that assignment IDs are unique for each Worker.</p><img alt="" height="1" src="https://medium.com/_/stat?event=post.clientViewed&amp;referrerSource=full_rss&amp;postId=245c6c3323bb" width="1" /><hr /><p><a href="https://blog.mturk.com/paying-for-non-submitted-hits-245c6c3323bb">Paying for Non-Submitted HITs</a> was originally published in <a href="https://blog.mturk.com">Happenings at MTurk</a> on Medium, where people are continuing the conversation by highlighting and responding to this story.</p>
